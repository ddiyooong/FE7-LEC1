mocacong: 코딩하기 좋은 카페

기획, 디자인, 안드로이드 담당

#프로젝트 개요

모카콩(mocacong)은 개발자들이 코딩에 집중할 수 있는 최적의 카페를 찾도록 도와주는 앱입니다. 함께 코딩할 동료를 찾고 싶은 개발자, 조용한 환경에서 혼자 집중하고 싶은 개발자 모두를 위해 카페의 다양한 리뷰와 댓글을 제공하며, 지도를 통해 주변의 코딩하기 좋은 카페를 한눈에 확인할 수 있도록 설계되었습니다.

#Firebase Crashlytics 기반 오류 추적 및 유지보수 자동화

모카콩 앱의 안정성 향상을 위해 Firebase Crashlytics를 도입하였습니다.
Crashlytics에서 제공하는 실시간 예외 로그를 기반으로 오류 발생 위치를 정확히 추적하고, 빈도 높은 오류에 대한 자동 테스트를 구축하여 앱의 장애 대응 속도를 개선하였습니다.
그 결과 RecyclerView의 IndexOutOfBoundsException 및 네트워크 예외 처리 로직을 개선하고, 버전 업데이트 이후 앱 크래시율을 40% 이상 감소시킬 수 있었습니다.

#네이버 지도 SDK 기반 위치 기반 카페 탐색 기능 구현

사용자 주변의 코딩하기 좋은 카페를 탐색할 수 있도록 네이버 지도 SDK for Android를 활용하였습니다.
FusedLocationProviderClient를 통해 현재 위치를 받아오고, 해당 위치를 기준으로 지도에 마커를 표시하여 카페 정보를 시각화했습니다.
마커 클릭 시 InfoWindow를 통해 요약 정보를 제공하고, 상세 페이지로 연결되는 흐름을 구성하였습니다.
지도 줌 레벨 변화에 따라 자동으로 리스트가 갱신되도록 구현하여, 사용자가 반경 5km 내 추천 카페를 직관적으로 탐색할 수 있게 했습니다.

#MVVM 아키텍처 기반 앱 구조 설계

모카콩 앱은 MVVM 아키텍처를 기반으로 개발되어 유지보수성과 테스트 편의성을 고려했습니다.
ViewModel에서 UI 로직을 분리하고, Repository 계층에서는 Firebase 및 Room DB의 데이터를 통합 관리하도록 구성하였습니다.
기존의 단일 Activity 중심 구조를 분리하여 각 화면을 독립적으로 테스트 가능하도록 구성하였으며, 이를 통해 코드 가독성과 확장성을 확보하였습니다.

#RecyclerView + DiffUtil을 활용한 리뷰 리스트 최적화

리뷰 목록 화면의 성능을 개선하기 위해 DiffUtil.ItemCallback을 활용하여 변경된 항목만 효율적으로 갱신하도록 구현했습니다.
이전에는 리뷰 추가 또는 수정 시 전체 리스트를 새로 그리는 방식이었으나, DiffUtil 적용 이후 평균 60fps 이상의 성능을 유지할 수 있었습니다.
또한, 각 리뷰 항목에는 콘센트 유무, 와이파이 속도, 소음도 등 코딩 환경 정보를 시각화하여 사용자 경험을 강화하였습니다.

#다크모드 및 접근성 대응

사용자 편의를 위해 다크모드를 지원하고 접근성 가이드라인에 따라 UI를 개선하였습니다.
values-night 리소스를 별도로 구성하여 어두운 테마에 적합한 색상을 지정하고, 텍스트 대비를 조절하여 가독성을 확보하였습니다.
모든 이미지 요소에 contentDescription을 추가하고, 버튼 및 터치 영역을 최소 48dp 이상으로 설정하여 스크린 리더 및 저시력 사용자도 편리하게 사용할 수 있도록 대응하였습니다.
이러한 개선을 통해 사용자 리뷰 중 UI 만족도가 20% 이상 향상되었습니다.
