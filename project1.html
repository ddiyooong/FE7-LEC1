#프로젝트 개요

모카콩(mocacong)은 개발자들이 코딩에 집중할 수 있는 최적의 카페를 찾도록 도와주는 앱입니다. 함께 코딩할 동료를 찾고 싶은 개발자, 조용한 환경에서 혼자 집중하고 싶은 개발자 모두를 위해 카페의 다양한 리뷰와 댓글을 제공하며, 지도를 통해 주변의 코딩하기 좋은 카페를 한눈에 확인할 수 있도록 설계되었습니다.

#Firebase Crashlytics 기반 오류 추적 및 유지보수 자동화

앱을 배포한 후, 다양한 사용자 환경에서 예외 상황이 자주 발생하는 것을 확인했습니다.
테스트 환경에서는 재현되지 않는 문제가 많았고, 이로 인해 유지보수가 어려웠습니다.
안정적인 운영을 위해 여러 오류 추적 솔루션을 검토했으며, 그중 실시간 수집과 통계 기능이 강력한 Firebase Crashlytics를 도입하기로 결정했습니다.
도입 후에는 오류 발생 위치와 원인을 빠르게 파악할 수 있었고, 이를 기반으로 네트워크 예외 처리와 RecyclerView 예외에 대한 수정 작업을 신속히 수행했습니다.
Crashlytics를 통해 앱 크래시율을 40% 이상 감소시킬 수 있었고, 이 경험을 통해 운영 환경에서의 관찰과 자동화의 중요성을 느꼈습니다.

#네이버 지도 SDK 기반 위치 기반 카페 탐색 기능 구현

앱의 핵심 기능 중 하나는 ‘내 주변 코딩하기 좋은 카페’ 추천이었습니다.
초기에는 단순한 리스트 형태였지만, 위치 기반 정보 없이는 직관적인 탐색이 어렵다는 피드백이 있었습니다.
Google Maps, Kakao Maps, Naver Maps SDK를 비교한 결과, 마커 커스터마이징과 국내 사용 편의성이 높은 네이버 지도 SDK를 선택했습니다.
지도에 현재 위치 기반 마커를 띄우고, 클릭 시 요약 정보와 상세 페이지로 연동하는 구조로 개선하였습니다.
사용자는 반경 5km 내 카페를 직관적으로 탐색할 수 있었고, 개발자로서 사용자 위치 UX의 핵심은 정보량보다 탐색 흐름의 자연스러움이라는 점을 체감했습니다.

#MVVM 아키텍처 기반 앱 구조 설계

프로젝트 초기에는 로직과 UI 코드가 섞여 있어 유지보수에 어려움이 있었습니다.
특히 화면 간 상태 공유와 테스트 코드 작성이 비효율적이라는 점이 문제였습니다.
처음에는 가장 간단하다고 하는 MVC패턴을 적용하였는데, 안드로이드 생태계 특성 상 오히려 복잡해지는 코드가 많았습니다. 따라서 안드로이드에서 권장하는 MVVM 아키텍처를 선택하게 되었습니다.
ViewModel을 중심으로 비즈니스 로직을 분리하고, Repository에서 Firebase 및 로컬 DB를 통합 관리하는 구조로 리팩토링하였습니다.
그 결과, 화면별 단위 테스트가 가능해졌고 기능 변경 시 파급 범위가 줄어들어 유지보수성이 크게 향상되었습니다.

#RecyclerView + DiffUtil을 활용한 리뷰 리스트 최적화

카페 리뷰는 사용자 참여도가 높은 기능이었지만, 리뷰 작성 후 리스트 전체가 갱신되면서 스크롤 튐 현상이 발생했습니다.
리스트의 갱신 방식 자체에 문제가 있다고 판단하여, DiffUtil.ItemCallback을 활용한 항목별 업데이트 방식을 도입했습니다.
다양한 방법을 검토했지만, LiveData 또는 notifyDataSetChanged() 중심의 기존 방식은 성능 저하를 유발했기 때문에, 가장 효율적인 선택은 DiffUtil이었습니다.
이후에는 평균 60fps 이상의 성능을 유지하며, UI가 부드럽게 유지되었고, 사용자의 체감 성능도 긍정적인 반응을 얻었습니다.
이 과정을 통해, 사용자 경험은 단순히 기능 제공이 아니라 반응성과 안정성까지 포함된 개념임을 실감할 수 있었습니다.
